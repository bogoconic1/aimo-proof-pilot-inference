# IMO 2026 — GPT-5.6 Solutions + Markschemes

Six olympiad problems in the IMO 2026 mold (6 problems, 7 points each), each with a
model-written reference solution and a per-problem marking scheme, generated with
GPT-5.6.

## Fields

Each JSONL record has:

| field | description |
|---|---|
| `problem_idx` | `"1"`–`"6"` |
| `problem` | full problem statement (LaTeX) |
| `reference_solution` | a complete worked proof; the determine/find-all problems lead with the boxed answer |
| `points` | `7` |
| `grading_scheme` | list of `{title, points, desc}` rubric items; the points sum to 7 |

## ⚠️ Important: the markschemes are reference-anchored

The `grading_scheme` items are **milestone rubrics written around the specific
reference solution**, not fully method-agnostic "grade any solution" instructions.
The item *titles* are largely universal (facts any solution must establish), but the
*descriptions* describe the reference route, and the "or equivalent" allowance is
sparse.

Practical consequence — especially for an **LLM-as-judge** pipeline:

- **P1, P3, P5, P6** grade reasonably well across methods (their milestones are the
  boxed answer and forced lemmas that essentially any solution must reach).
- **P2 (geometry)** and **P4** are tied to the reference approach. A correct proof by
  a *different* method (e.g. trigonometric / complex / barycentric for P2, or a
  different winning strategy for P4) can hit few of the listed milestones and be
  under-scored.

If you use these to grade arbitrary candidate proofs, instruct the grader to award
each item for the **underlying mathematical fact achieved by any rigorous route**, not
only for the reference construction.

## Provenance

- **Problems**: IMO-2026-labeled; character names are renamed to Chinese
  historical/legendary figures (Confucius, Liu Bang, Xiang Yu, Shan-Yu, Mulan). Not
  independently verified as the verbatim official problems.
- **Solutions + markschemes**: generated by GPT-5.6.
- **Split**: `train` (6 records).
